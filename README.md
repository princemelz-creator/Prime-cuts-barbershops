# PRIME CUTS BABERSHOPS

A professional wed presence for **Prime Cuts Barber Shops**,a premium barber shop specializing in classic grooming and modern styling.


## Project Overview


This project is a multi-page static website designed to showcase shop services, share grooming expertise via a blog, and facilitate customer booking. 


##  Project Structure
- `index.html` : The landing page featuring a hero section and service summary.
- `about.html` : this is detailed information about the shops history and team skill set.
- `contact.html` : it entails the location, working hours and contact information where you can call for more info.
- `blog.html` : A content field for grooming tips and styles.
- `gallery.html`: A filed that contains our company pictures and are displayed in this page.
- `service.html`: A content filed that details all the services provided clearly.

## Getting started
1. Clone this repository on your machine.

```bash
git clone https://github.com/princemelz-creator/Prime-cuts-barbershops.git

cd prime-cuts-barber-shops
```
2. Open `index.html` in any modern browser to view the wed page.
3. No build tools or dependencies required for this version.

## How to collaborate 
We Welcome contributions! To maintain code quality and project integrity, please follow the steps

### The workflow
- **Fork the Repository:** Create your own copy for the project to add on.
- **Create a Feature branch:**
```bash
git checkout -b feature/YourFeatureName
```
-**Commit Your Changes:** Use descriptive commit message (e.g.,feat: add social media icons to footer).
- **Push to branch:**
```bash
git push origin feature/YourFeatureName
```
- **Open a Pull Request(PR):**
Describe your changes clearly and link the issues found.

2. **ConTRIbution Guidelines**
Maintain Semantics: keep the HTML structure semantic. Avoid using
`<div>` where `<section>`, or <nav> is more appropriate.

Code Style: Ensure all tags are properly closed and indented using 2 or 4 spaces.

Documentation: If you add a new page or a significant feature,
update the "Project Structure" section of the README.

3. Reporting Issues
If you find a mistake or a suggestion:

Check the Issues tab to see if it has already been reported.

If not, open a new issue with a clear title and a detailed description of the problem or feature request.


---

### Why this structure works for your team:
* **Branch Naming;** By suggesting `feature/YourFeatureName` , you prevent the main branch from becoming a "junkyard" of half-finished code.
* **Pull Request:** This allows you, as the lead it goes live,ensuring the "Prime Cuts" brand stays high quality.
* **Commit Messages:** Using prefixes like `feat` or `fix:` makes it much easier to scan the project history later on.
